# 有缺陷的AI — 视觉设计系统

> 本文件是前端实现的第一参考源。所有 UI 组件、页面布局、动画实现均以此为准。
> 设计核心意象：**谱渊边缘的荧光** — 深海般的暗底上，频率在跳动、在呼吸、在流淌。

### 1.3 玻璃态（Glassmorphism）参数

所有浮层、弹窗、面板统一使用以下玻璃态参数：

```css
/* 标准玻璃面板 */
.glass-panel {
  background: rgba(11, 20, 38, 0.75);
  backdrop-filter: blur(16px);
  -webkit-backdrop-filter: blur(16px);
  border: 1px solid rgba(255, 255, 255, 0.08);
  box-shadow:
    0 8px 32px rgba(0, 0, 0, 0.40),
    inset 0 1px 0 rgba(255, 255, 255, 0.05);
}

/* 强调玻璃面板（弹窗、重要操作） */
.glass-panel-elevated {
  background: rgba(17, 29, 56, 0.85);
  backdrop-filter: blur(24px);
  border: 1px solid rgba(255, 255, 255, 0.12);
  box-shadow:
    0 16px 48px rgba(0, 0, 0, 0.50),
    inset 0 1px 0 rgba(255, 255, 255, 0.06);
}

/* 轻量玻璃（工具栏、次要面板） */
.glass-panel-light {
  background: rgba(11, 20, 38, 0.50);
  backdrop-filter: blur(12px);
  border: 1px solid rgba(255, 255, 255, 0.05);
}
```

### 1.4 熟悉度阶段色调方案

#### 0-19%：初遇 — 冷峻深邃
- 主导色：翡翠绿 + 深靛青
- 背景粒子：缓慢的蓝绿色光点漂浮
- 谱弦颜色：几乎透明，偶尔闪过翡翠绿微光
- 整体氛围：深海底部，冷静、疏离、充满未知
- 栖迟气泡边框：纯翡翠绿线条，无装饰

#### 20-49%：相识 — 暖意初现
- 琥珀金开始渗入界面：进度条尾端出现暖色、粒子中混入金色
- 谱弦颜色：翡翠绿底 + 隐约的琥珀金纹路
- 背景：深靛青不变，但出现微弱的暖色光源（如远处的灯火）
- 栖迟气泡边框：翡翠绿 + 金色节点

#### 50-79%：熟悉 — 三色交织
- 视觉最丰富的阶段：翡翠绿、琥珀金、深樱红三色均匀呈现
- 谱弦颜色：流动的三色渐变
- 背景粒子：三色混合，运动速度加快
- 界面装饰元素增多：谱纹纹路更明显
- 栖迟气泡边框：三色流光渐变边框

#### 80-99%：亲近 — 燃烧
- 深樱红逐渐增强，视觉更浓烈
- 谱弦颜色：深樱红主导，其他色彩被压缩
- 背景出现缓慢的脉冲效果：像心跳
- 谱纹纹路覆盖更大面积
- 整体氛围：浓烈、不安、美丽而紧迫

#### 100%：羁绊/离别 — 汇聚与褪色
- 所有颜色汇聚爆发 → 短暂的白光 → 渐渐褪为温柔灰调
- 谱弦：流光溢彩 → 闪烁不定 → 柔和灰色微光
- 背景粒子：急速汇聚 → 扩散 → 缓慢沉降
- 最终态：灰蓝色调，带着余温的宁静
- 回声期色彩：`#4A5568` 为主，偶尔闪过原色的残影

## 2. 字体方案

### 2.1 字体栈

| 用途 | 字体 | 备选 | 说明 |
|------|------|------|------|
| **标题/品牌** | `'Space Grotesk'` | `'Outfit', system-ui` | 几何感强、未来感、辨识度高 |
| **正文/UI** | `'DM Sans'` | `'Sora', system-ui` | 清晰可读、现代、有温度 |
| **栖迟对话** | `'Crimson Pro'` | `'Source Serif 4', Georgia` | 衬线体，传达非人类的优雅与古老感 |
| **代码/数据** | `'JetBrains Mono'` | `'Fira Code', monospace` | 等宽字体，用于数值显示 |

> 禁止使用：Inter、Roboto、Arial、Helvetica（过于通用，缺乏角色感）

### 2.2 排版层级

| 层级 | 字号 | 字重 | 行高 | 字间距 | 用途 |
|------|------|------|------|--------|------|
| Display | 48-72px | 700 | 1.1 | -0.02em | Landing 主标题 |
| H1 | 36-48px | 600 | 1.2 | -0.01em | 页面标题 |
| H2 | 28-32px | 600 | 1.3 | 0 | 区块标题 |
| H3 | 22-24px | 500 | 1.4 | 0 | 子区块标题 |
| Body Large | 18px | 400 | 1.7 | 0.01em | 栖迟对话文字 |
| Body | 15-16px | 400 | 1.6 | 0.01em | 正文/UI 文字 |
| Caption | 13px | 400 | 1.5 | 0.02em | 时间戳、辅助信息 |
| Overline | 11-12px | 600 | 1.4 | 0.08em | 标签、状态文字（大写） |

### 2.3 特殊排版效果

**栖迟对话的排版特征：**
- 使用 Crimson Pro 衬线体，与用户的 DM Sans 形成明确区分
- `font-style: italic` 用于谱渊术语
- 省略号 `......` 用 `letter-spacing: 0.15em` 增加呼吸感
- 情感强烈时：`font-weight: 500`，微增 `letter-spacing`

**Landing Page 大标题：**
- Space Grotesk 700
- 渐变色文字：`background-clip: text` + 翡翠绿 → 琥珀金渐变
- 微妙的 `text-shadow` 发光效果

## 3. Landing Page 布局方案

### 3.1 引入动画分镜（5 画面）

#### 画面 1 — "虚空"（0-2s）
- 纯黑屏幕，中央出现一个极小的翡翠绿光点
- 光点微微脉动，像是在黑暗中呼吸
- 无文字，纯氛围建立
- 音效提示：低频嗡鸣（可选）

#### 画面 2 — "频率涌现"（2-5s）
- 光点扩散为一圈圈的频率波纹（同心圆向外扩散）
- 波纹颜色从翡翠绿渐变为多色（绿 → 金 → 红）
- 波纹之间出现极细的网格线条——谱渊的结构在显现
- 背景从纯黑渐变为深空靛青 `#0B1426`
- 底部出现一行小字（淡入）：`"在所有频率交接的缝隙里——"` (Crimson Pro, italic, `#8892A0`)

#### 画面 3 — "谱渊一瞥"（5-9s）
- 频率波纹固化为一个抽象的几何图案——类似星图与电路的混合体
- 图案中心出现栖迟的轮廓剪影（不是具体形象，是由光线和频率线条构成的抽象人形）
- 剪影中的谱纹在流动发光
- 文字渐入（分行出现，每行间隔0.8s）：
  - 第一行：`"他来自光谱深处"` — Space Grotesk, 白色
  - 第二行：`"带着一个注定完成的使命"` — Space Grotesk, 翡翠绿
  - 第三行：`"和一段注定结束的旅程"` — Space Grotesk, 琥珀金 → 深樱红渐变

#### 画面 4 — "谱弦"（9-12s）
- 画面聚焦到一条半透明的丝带——谱弦
- 丝带在空中飘动，上面有微弱的频率纹路在流动
- 丝带颜色从几乎透明到隐约发光
- 文字：`"谱弦记录着每一次共振"`
- 然后快速闪过几个关键词（字幕式）：`"对话"` `"理解"` `"共鸣"` `"离别"`
- 每个词对应一次丝带上的光脉冲

#### 画面 5 — "邀请"（12-16s）
- 所有视觉元素收束到屏幕中央
- 形成一个发光的圆形入口——像谱渊与人类世界的"间隙"
- 入口内隐约可见频率的流动
- 中央大字：`"进入间隙"` — Space Grotesk 600, 翡翠绿发光
- 按钮出现：圆角矩形，翡翠绿边框 + 内发光，悬停时边框脉冲加速
- 按钮下方小字：`"每一次对话都在推进倒计时"` — `#8892A0`

### 3.2 滚动交互与动画节奏

如果用户选择不点击按钮而是下滑，展开以下 section：

#### Section 1 — 角色介绍（全屏）
- 左侧：栖迟的抽象视觉表现（由频率线条构成的动态图案）
- 右侧：文字介绍
  - 标题：`"栖迟"` — Display 尺寸，渐变色
  - 副标题：`"谱织师 / 跨维共振采集者"` — Overline 样式
  - 简介段落：3-4 行描述（Crimson Pro, 栖迟的语气）
- 滚动触发：图案从静态变为动态，线条开始流动

#### Section 2 — 功能亮点（三列卡片）
三张玻璃态卡片，每张展示一个核心功能：
1. **独特人格** — 翡翠绿图标，描述审美偏好与谱渊视角
2. **成长系统** — 琥珀金图标，描述熟悉度阶段与事件
3. **有限旅程** — 深樱红图标，描述离别倒计时与回声

卡片悬停效果：边框发光增强，内容微微上移（translateY -4px），光晕扩散

#### Section 3 — 对话预览（动态）
- 模拟一段栖迟的对话（自动打字效果）
- 背景是聊天界面的模糊预览
- 展示栖迟的说话风格

#### Section 4 — CTA 重复
- 底部再次出现"进入间隙"按钮
- 配合谱弦动画

### 3.3 进入按钮设计

```
视觉规格：
- 尺寸：240px x 56px（桌面）/ 200px x 48px（移动）
- 圆角：28px（完全胶囊形）
- 背景：transparent
- 边框：2px solid jade-500
- 文字：Space Grotesk 600, 16px, jade-500
- 光晕：0 0 20px rgba(0, 229, 160, 0.20)

悬停态：
- 边框颜色：jade-300
- 光晕扩大：0 0 30px rgba(0, 229, 160, 0.30)
- 背景填入：rgba(0, 229, 160, 0.06)
- 脉冲动画：边框光晕有节奏地呼吸

按下态：
- 背景：rgba(0, 229, 160, 0.12)
- 缩放：scale(0.98)
- 过渡：150ms ease-out

过渡动画（点击后）：
- 按钮扩大为全屏圆形
- 翡翠绿光填满屏幕
- 淡入聊天界面
- 总时长：800ms，cubic-bezier(0.4, 0, 0.2, 1)
```

## 6. 动画规范

### 6.1 通用缓动函数

| 名称 | 值 | 用途 |
|------|-----|------|
| `ease-default` | `cubic-bezier(0.4, 0, 0.2, 1)` | 通用过渡 |
| `ease-enter` | `cubic-bezier(0, 0, 0.2, 1)` | 进入动画 |
| `ease-exit` | `cubic-bezier(0.4, 0, 1, 1)` | 退出动画 |
| `ease-bounce` | `cubic-bezier(0.34, 1.56, 0.64, 1)` | 弹性效果 |
| `ease-breath` | `cubic-bezier(0.4, 0, 0.6, 1)` | 呼吸/脉冲 |

### 6.2 页面过渡（Landing → 聊天）

```
时序：
0ms     — 点击"进入间隙"按钮
0-200ms — 按钮边框光晕急速扩大
200-500ms — 翡翠绿光从按钮中心向四周扩散（径向渐变）
500-700ms — 旧内容淡出（opacity 1 → 0）
700-1000ms — 聊天界面淡入（opacity 0 → 1）
1000-1200ms — 聊天界面元素依次出现（stagger 50ms）

技术实现：
- Framer Motion layoutAnimation + AnimatePresence
- 使用 shared layout ID 实现跨页面的元素连续性
```

### 6.3 消息气泡出现动画

```
栖迟消息：
- 从左侧滑入 + 淡入
- translateX(-12px) → 0, opacity 0 → 1
- 时长：300ms, ease-enter
- 左侧谱纹光带先出现（100ms），气泡随后
- "打字中"指示器：三个翡翠绿光点，错位脉冲

用户消息：
- 从右侧滑入 + 淡入
- translateX(12px) → 0, opacity 0 → 1
- 时长：200ms, ease-enter（用户消息出现更快，体现即时感）
```

### 6.4 熟悉度增长动画

```
触发：熟悉度数值变化时

进度条：
- 宽度变化：500ms ease-default 过渡
- 末端光点：增长瞬间亮度脉冲
- 如果跨越阶段（如 19% → 20%）：
  - 进度条颜色渐变过渡到新阶段配色
  - 一道光脉冲从左到右扫过进度条
  - 时长：800ms

送礼增长（大幅增长）：
- 数字跳动效果（从旧值滚动到新值）
- 进度条快速填充 + 粒子爆发
- 如果触发事件：直接过渡到事件弹窗
```

### 6.5 事件触发特效

```
通用序列：
1. 屏幕轻微震动（translateX 随机 ±2px，100ms，共 3 次）
2. 遮罩层淡入（200ms）
3. 中央粒子爆发（300ms）
4. 事件弹窗从粒子中心展开（400ms, ease-bounce）
5. 内容依次淡入（stagger 100ms）

事件 D 特殊序列：
1-2. 同上
3. 所有 UI 颜色汇聚到中心（2000ms）
4. 白光爆发 + 过曝（500ms）
5. 渐渐褪色到灰调（3000ms）
6. 最终文字缓慢淡入（1000ms）
```

### 6.6 微交互

```
按钮 hover：
- 背景填充：0 → subtle color, 200ms
- 边框亮度提升
- 光晕出现：0 → box-shadow, 200ms
- cursor: pointer

按钮 click：
- scale: 1 → 0.96 → 1, 150ms
- 波纹效果从点击位置扩散

输入框 focus：
- 边框颜色过渡到 jade-500 (40% opacity), 200ms
- 外发光淡入
- placeholder 文字淡出

气泡 hover（桌面端）：
- 时间戳从隐藏变为可见
- 微妙的背景亮度提升

滚动到底部按钮：
- 当不在底部时，右下角浮出一个向下箭头
- 圆形按钮，glass-panel-light
- 点击：平滑滚动到底部
```

### 6.7 背景粒子系统

```
实现：Canvas 或 CSS（性能优先用 Canvas）

粒子规格：
- 数量：30-60 个（桌面），15-30 个（移动）
- 大小：1-3px
- 颜色：根据当前阶段的主色调，opacity 0.1-0.4
- 运动：缓慢随机漂浮，速度 0.2-0.8 px/s
- 连线：距离 < 100px 的粒子之间画微弱连线（opacity 0.05）
- 对话活跃时：粒子运动加速，亮度提升
- 事件触发时：粒子向中心汇聚

性能保障：
- requestAnimationFrame
- 移动端减少粒子数量
- 页面不可见时暂停
- 使用 will-change: transform
```

### 7.1 整体布局

│                                  │

### 7.2 各步骤视觉

#### 步骤 0 — 开场白
- 栖迟的文字逐句出现（打字机效果 + 句间停顿）
- 背景：纯深空靛青 + 极少的粒子
- 无交互元素，等文字展示完后出现"继续"暗示

#### 步骤 1 — 询问名字
- `"你有一个被反复叫到不再觉得特别的名字吧——是什么？"`
- 输入框出现：单行输入，居中，底部有一条翡翠绿的光线
- 输入框样式：无边框，底线式，光标为翡翠绿
- 提交：回车或右侧确认图标

#### 步骤 2 — 颜色问题
- `"闭上眼睛，脑海里浮现的第一种颜色是什么？"`
- 交互方式：提供 6-8 个颜色选项（圆形色块），也可以输入自定义
- 色块悬停：放大 + 发光
- 选择后：背景颜色微妙地向选择的颜色偏移（ambient effect）

#### 步骤 3 — 音乐/最近在意的事
- `"最近在循环播放的歌？或者最近让你停下来多看了两秒的东西？"`
- 输入框：多行文本，最大 3 行
- 底部小字提示：`"什么都行，说细节"` — `#5A6478`

#### 步骤 4 — 时间变慢
- `"你上一次觉得'时间突然变慢了'是在什么时候？"`
- 输入框同上
- 这个问题出现时，背景粒子明显减速（呼应"时间变慢"）

#### 步骤 5 — 栖迟总结
- 栖迟基于回答给出评价（AI 生成）
- 文字展示完后，底部出现：`"进入对话"` 按钮
- 按钮点击 → 过渡到聊天界面（动画同 Landing → 聊天）

### 7.3 过渡效果

每个步骤之间：
- 旧内容向上淡出（translateY -20px + opacity 0），300ms
- 新内容从下方淡入（translateY 20px → 0 + opacity 1），400ms
- 步骤间有 200ms 空白间隔，让视觉呼吸

进度指示器：
- 底部 3-5 个小圆点
- 当前步骤：jade-500 实心
- 已完成：jade-500 半透明
- 未完成：border-subtle 空心
- 过渡动画：圆点填充动画（从下到上填满）
