# 开发计划

> 本文件定义 agent team 角色分工与开发阶段。
> 所有设计细节以 `project/设计文档/项目圣经.md` 为唯一来源。

---

## Agent Team 角色

| # | 角色 | 职责 |
|---|------|------|
| 1 | **team-lead** | 总协调、进度管理、质量把关、git commit |
| 2 | **character-designer** | 角色深化、世界观扩展、Prompt 设计 |
| 3 | **ui-designer** | 视觉规范、配色、动画方案、页面布局 |
| 4 | **frontend-dev** | 页面与交互实现（必须先读 frontend-design skill） |
| 5 | **backend-dev** | API、Prompt 引擎、成长/存档/燃料系统逻辑 |
| 6 | **test-engineer** | 单元测试 + 集成测试 |

### 协作依赖

```
项目圣经（设计来源）
      ↓
character-designer（角色深化）
      ↓
ui-designer（视觉规范）──→ frontend-dev（代码实现）
      ↓                          ↓
backend-dev（系统逻辑）←──── 集成对接
      ↓
test-engineer（全程跟进）
```

## 开发阶段

### Phase 0：设计定稿 ✅ 已完成

**目标**：项目圣经完善到可以直接指导开发

- [x] 用户完成所有设计输入（角色建模、世界观细节、交互细节）
- [x] 角色深化：诗人的语言风格、好恶、各阶段行为变化
- [x] 角色深化：企鹅的变形触发条件、视觉描述
- [x] 系统量化：航程燃料增减公式、火种生成频率、失速阈值
- [x] 终局演出：双结局的完整分镜描述
- [x] 灯塔模式：具体交互规则
- [x] 自我介绍流程：具体对话设计
- [x] 事件内容：20%/50%/80%/100% 四个事件的剧情

### Phase 1：UI 视觉设计

**依赖**：Phase 0 完成

- [ ] 读取 frontend-design skill，确定美学方向
- [ ] 配色体系（基于新世界观，鲜艳实验风）
- [ ] 字体方案
- [ ] Landing Page 分镜（引入动画）
- [ ] 聊天界面布局
- [ ] 终局演出视觉方案（元素溶解/风化/海面）
- [ ] 灯塔模式界面
- [ ] 输出 `design-system.md` + `tailwind-tokens.md`

### Phase 2：核心系统重构 ← 当前阶段

**依赖**：Phase 0 完成（可与 Phase 1 并行）

- [ ] 更新 GameState 类型（新增燃料、终局、企鹅状态）
- [ ] 熟悉度系统（纯对话驱动，移除金币加成）
- [ ] 金币系统（仅用于企鹅变形/视觉效果）
- [ ] 企鹅变形系统
- [ ] 航程燃料系统
- [ ] 火种生成与判定系统
- [ ] 终局触发与状态切换
- [ ] Prompt 引擎（基于新角色设定）
- [ ] 单元测试

### Phase 3：前端实现

**依赖**：Phase 1 + Phase 2

- [ ] Landing Page（引入动画）
- [ ] 趣味自我介绍流程
- [ ] 聊天界面（消息、输入、送礼、顶部栏）
- [ ] 熟悉度/燃料进度条（80% 后视觉转换）
- [ ] 充值界面（6 选项，仅用于企鹅变形）
- [ ] 事件弹窗
- [ ] 终局演出（双结局动画）
- [ ] 灯塔模式界面

### Phase 4：系统集成

**依赖**：Phase 3

- [ ] 聊天 ↔ GLM-5（流式渲染）
- [ ] 对话 → 熟悉度 → 进度条
- [ ] 企鹅变形完整流程
- [ ] 终局触发 → 选择 → 演出
- [ ] 成为诗人后的完整交互链路（燃料/火种/失速）
- [ ] 存档/读档
- [ ] 集成测试

### Phase 5：打磨

- [ ] 动画流畅度、微交互
- [ ] Prompt 调优（实际对话体验）
- [ ] 响应式适配
- [ ] 全量测试通过
- [ ] 部署

---

## 运行规则

- 迭代上限 50 轮，每阶段完成后 `npm test` → `npm run lint` → `npm run build`
- 每个 Phase 完成后 git commit
- 任务反复失败（>3次）→ 跳过并记录到 `progress.md`
- rate limit 暂停等待，刷新后自动恢复
